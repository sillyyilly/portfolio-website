Grocery Website Case Study

Project Overview

Save On Foods is among many grocery stores that provide online shopping as an option to their customers. With many competitors, the need to show users the savings and deals that Save On Foods has to offer is imperative to company success. This is the need we chose to focus on when conducting this UX research study.

Responsibilities

  • Lead and conduct user research
  • UX analysis and facilitating the design thinking process

Tools

  • Figma
  • Adobe Illustrator
  • Adobe Photoshop

Project Goals

Our primary goals were to:

  • Gauge the discoverability/visibility of the deals tab and all the sub pages that are available to their use, in order to save money on their grocery orders.
  • Understand how much time is spent to find an item using the “Deals” page vs using the “Shop Aisles” page. We want to assess the learnability and efficiency of each page and whether or not one is easier to access than the other

The Process

We followed the double diamond UX process for this case study to define the problem clearly in order to produce the best solution for our users.The four stages of the double diamond process are Discovery, Definition, Development, and Delivery. These stages are a roadmap that designers use to organize the creative process when improving user experience.

The Discover Phase

Research Methods Used

  • Semi-stuctured interviews
  • Direct observation

User Interviews

Our user interviews were conducted in two parts: an observation of them performing a task, followed by a semi-structured interview. We wanted to compare the ease of finding an item under the “Shop Aisles” tab compared to finding an item under the “Deals” tab, and we did this by timing how long it took participants to find the same item using different pages, as well as recording the screen and tallying the number of clicks used.

For the semi-structured interview, we gathered qualitative data by asking about their experience, and quantitative data about which “Deals” subcategory they used the most. The data was gathered using coding sheets, then sorted using an affinity diagram.

The Define Phase

Methods Used

  • Affinity Diagrams
  • User personas

User Personas

After gathering the data, we generated personas off our statistical knowledge that represent certain types of users that are the audience of our solution. This allows us to fine-tune our design for our target users.

The Development Phase

Results for prototyping

The findings from the interviews are what we would use to help prototype the improved design.

  • The interface’s strength is the “Shop Aisles” page, as it took on average 29.6 seconds faster to find an item there compared to the “Deals” page
  • The interface’s weakness was the “Deals” page, which took on average 10 clicks to find an item, compared to 6 clicks using the “Shop Aisles” page
  • The filter function was broken on some of the pages, with the letters only going up to “c”

Design Requirements

The user interview results combined with the user personas we created allow for a comprehensive list of design requirements ready for implementation

  • The system must have a complete list of filters on the side of the “All Specials” page.
  • The system must have the regular price displayed next to the deal price.
  • The system must be accessible for users via the internet (both mobile and desktop).
  • The system should remove the ads at the top of its “Save on, Savers” page.
  • The system could make the titles of the deals pages more intuitive.
